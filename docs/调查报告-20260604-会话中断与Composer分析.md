## 二、Composer-2.5-Fast 附带调用分析

### 2.1 现象

使用 `claude-4.6-opus-high-thinking` 时，用量面板同时显示了 `composer-2.5-fast` 的 17.9万 tokens 消耗。

### 2.2 原因

这是 **Cursor 的预期行为**。Cursor 内置的子代理 (Subagent) 系统会自动创建辅助代理：

| 子代理类型 | 功能 | 默认模型 |
|-----------|------|---------|
| **Explore** | 并行搜索代码库、查找文件、收集上下文 | Composer 2 Fast |
| **Bash/Shell** | 执行 shell 命令 | Composer 2 Fast |
| **Browser** | 浏览器自动化 | Composer 2 Fast |

主 Agent 使用用户选择的模型（Opus 4.6），子代理默认使用 Composer 2 Fast。

### 2.3 对按次数计费 (Legacy Plan) 的影响

- **每个子代理调用会消耗一个独立的 request 额度**
- 一次主请求可能触发 1-2 个额外的子代理 request
- 子代理的探索工作完全可以由主 Agent 自己完成（串行执行）

### 2.4 控制方式

| 方式 | 效果 | 可靠性 |
|------|------|--------|
| Settings > Agents > Subagents → `Inherit from parent` | 子代理改用主模型，仍消耗额外 request | 中 |
| `.cursor/rules/` 添加规则 `Don't run subagents` | 软禁止，模型通常遵守 | 低-中 |
| `subagentStart` Hook + `permission: "deny"` | 代码层面硬禁止 | 高 |

### 2.5 建议

对按次数计费用户：**建议禁止子代理**，Opus 4.6 自身有完整的文件读取、搜索、shell 执行能力，关闭子代理仅牺牲少量并行速度，但节省 request 配额。
