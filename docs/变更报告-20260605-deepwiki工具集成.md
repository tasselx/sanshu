## 功能特性

### 三种操作模式

| 操作 | 说明 | 典型用途 |
|------|------|----------|
| `structure` | 获取仓库文档结构 | 了解仓库整体架构 |
| `content` | 获取章节详细内容 | 深入阅读某个模块的文档 |
| `ask` | 智能问答 | 对仓库提出具体问题 |

### 技术实现

- **协议**：JSON-RPC 2.0 over HTTP（streamable-http）
- **端点**：`https://mcp.deepwiki.com/mcp`
- **响应格式**：支持 JSON 和 SSE 两种模式
- **认证**：无需认证，免费使用
- **初始化**：首次调用自动完成 MCP initialize 握手

## context7 + deepwiki 配合策略

| 策略 | 描述 |
|------|------|
| **基础分工** | context7 = 官方 API 文档；deepwiki = 仓库架构与源码理解 |
| **优先级降级** | 常见框架先用 context7，查不到再用 deepwiki；小众库直接 deepwiki |
| **冲突解决** | 两者信息矛盾时，以 context7（官方文档）为准 |
| **深度调试** | 遇到框架 bug/异常行为时，用 deepwiki 看源码实现定位根因 |
| **选型对比** | 用 deepwiki 对比多个候选库的架构质量和设计理念 |
| **迁移辅助** | context7 查迁移指南 + deepwiki 查 breaking changes 源码 |
| **PR/Issue 研究** | 用 deepwiki ask 功能研究仓库的设计决策和历史背景 |
